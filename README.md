# 分光光度模块串口记录工具

本工具用于通过 Windows 串口读取分光光度模块返回的 24 路数据，并保存为 Excel。

当前入口脚本为：

```text
app.py                 图形界面入口
```

当前代码结构：

```text
app.py                   图形界面启动入口
old/read_filter_stable.py 旧命令行脚本归档，不作为当前入口
config_loader.py         config.ini 读取、默认配置和配置值转换
app_paths.py             源码运行目录和 EXE 运行目录识别
serial_protocol.py       串口配置、命令构造、数据帧解析
filter_logic.py          波长映射、通道组、空气基底和稳定判定常量
excel_writer.py          输出目录、原始数据 CSV 缓存、原始数据 Excel、稳定值 Excel
spectro_ui/              PySide6 图形界面、通道卡片、后台串口线程
config.ini               默认运行配置
requirements.txt         Python 依赖列表
```

## 功能概览

- 提供 PySide6 图形界面，支持一页显示 24 个通道。
- 通过串口发送指定波长启动命令。
- 解析模块返回的 `5A A5 01 56 60 ...` 数据帧。
- 每收到一帧有效数据，就追加写入原始 CSV 缓存，停止后生成原始数据 Excel。
- 支持空气值记录模式。
- 支持 10%、20%、30% 滤光片测试模式。
- 滤光片模式会先建立空气基底，再按比例判断滤光片是否插入。
- 顶部显示实时进度，例如 `已写入 160 行，稳定 1/24`；选择 12 通道组时显示 `稳定 1/12`。
- 右侧显示运行日志。
- 通道卡片右上角状态点：灰色=未建立基底，绿色=可测试，黄色=检测到滤光片且正在等待 10 帧稳定，蓝色=稳定值已记录且等待拔出并回到空气。
- 稳定值保存为单独 Excel，并统计最小值、最大值、平均数、CV。

## 串口配置

串口配置已固定，不需要命令行传入：

```text
115200 8N1
数据位: 8
校验位: None
停止位: 1
```

默认串口号在 `config.ini` 中配置为 `COM6`。

## 波长与命令

脚本使用 A 组命令启动采集。以 520nm 为例，发送：

```text
A5 5A 01 50 02 02 01 55
```

波长编号对应关系：

```text
410nm -> 编号0 -> 紫色光
460nm -> 编号1 -> 蓝色光
520nm -> 编号2 -> 绿色光
550nm -> 编号3 -> 绿色光
590nm -> 编号4 -> 橙色光
630nm -> 编号5 -> 红色光
```

当前实测新板子中，A 组 `0x50` 命令会返回 24 路数据；B 组通常不需要单独打开。

## 安装依赖

需要 Python 以及以下库：

```powershell
pip install -r requirements.txt
```

## 配置文件

程序会读取同目录下的 `config.ini`。源码运行时是脚本所在目录；打包成 exe 后是 exe 所在目录。

默认配置示例：

```ini
[settings]
port = COM6
wavelength = 520
channel_group = 0
filter_ratio = 30
output = 串口数据记录.xlsx
stable_output = 滤光片稳定值.xlsx
ratio_tolerance = 5
air_tolerance = 5
no_start = false
keep_light = false
```

## 图形界面用法

启动 GUI：

```powershell
python app.py
```

界面流程：

1. 选择串口、波长、滤光片比例和通道组。
2. 点击 `开始测试`。
3. 保持空气状态，等待目标通道卡片右上角圆点变为绿色。
4. 插入滤光片，命中比例后通道圆点变为黄色。
5. 稳定后日志会显示稳定值，通道圆点变为蓝色。
6. 拔出滤光片，回到空气并稳定后，通道圆点重新变为绿色。
7. 点击 `停止`，程序会发送关闭命令并保存结果。

右侧只显示日志；实时行数和稳定数量显示在顶部。

## 打包 EXE

图形界面打包入口是 `app.py`：

```powershell
pyinstaller --onefile --windowed --name 分光光度滤光片记录 --add-data "assets;assets" app.py
```

打包完成后，把 `config.ini` 放到 exe 同目录：

```powershell
Copy-Item .\config.ini .\dist\config.ini -Force
```

窗口和任务栏图标使用 `assets\app_icon_spectro_transparent.png`。如果只复制 exe 运行，也可以把 `assets` 文件夹放在 exe 同目录。

如果需要 Windows 资源管理器里的 exe 文件本身也显示自定义图标，需要额外准备 `.ico` 文件并在打包时添加 `--icon 图标路径.ico`。

## 配置项说明

```text
port              串口号
wavelength        波长，可选 410/460/520/550/590/630
channel_group     测试通道组，0=1-24，1=1-12，2=13-24
output            原始数据 Excel 文件名
stable_output     稳定值 Excel 文件名
filter_ratio      0/10/20/30；0 表示只记录空气值
ratio_tolerance   滤光片比例允许误差百分点
air_tolerance     回到空气基底的允许误差百分点
no_start          true 表示不发送开灯命令，只读取已有数据流
keep_light        true 表示退出时不发送关灯命令
```

如果只接了一个 12 通道模块，要把 `channel_group` 设置为 `1` 或 `2`，否则空气基底阶段会等待未连接通道。

## 保存路径

程序会按运行时间创建记录目录，固定保存在程序所在目录下：

```text
程序目录\data\record_HHMMSS\
```

例如 14:00:00 运行：

```text
data\record_140000\
```

空气模式只保存：

```text
_temp_csv\串口数据记录.csv
串口数据记录.xlsx
```

滤光片模式保存：

```text
_temp_csv\串口数据记录.csv
串口数据记录.xlsx
滤光片稳定值.xlsx
```

## 原始数据文件格式

运行中每收到一帧有效串口数据，就先追加到 `_temp_csv\串口数据记录.csv`。停止后程序会根据 CSV 生成原始数据 Excel：

```text
时间 | CH1 | CH2 | ... | CH24
```

如果选择 `CH1-CH12` 或 `CH13-CH24`，原始 CSV 和 Excel 只保存当前选择通道组，不保存未选中的通道列。

第一列是当前时分秒，例如：

```text
15:25:03
```

所有 Excel 字体设置为 `微软雅黑`。

## 稳定值文件格式

稳定值文件格式：

```text
通道    第1次    第2次    第3次    ...    最小值    最大值    平均数    CV
CH1     数值     数值     数值            数值      数值      数值      数值
CH2     数值     数值                     数值      数值      数值      数值
...
CH24    数值     数值     数值            数值      数值      数值      数值

最小值  数值     数值     数值
最大值  数值     数值     数值
平均数  数值     数值     数值
CV      数值     数值     数值
```

同一个通道多次放入滤光片时，新稳定值会横向追加到右侧，不会覆盖之前的稳定值。右侧统计值按单个通道横向计算，用于查看该通道多次测试的稳定性；底部统计值按每一次测试纵向计算。  
如果选择 `CH1-CH12` 或 `CH13-CH24`，稳定值文件只保存当前选择通道组；统计值也只统计当前选择通道组中已经记录到稳定滤光片值的通道。

## 滤光片测试流程

滤光片模式下，流程如下：

1. 发送开灯命令。
2. 保持空气状态。
3. 脚本先采集目标通道空气基底。
4. 每个目标通道空气基底都满足稳定条件后，统一显示空气基底。
5. 清空基底阶段串口数据。
6. 开始正式采样并写入原始数据文件。
7. 插入滤光片。
8. 当 `当前值 / 空气基底` 命中指定滤光片比例，打印检测提示。
9. 命中比例后，连续稳定 10 帧，写入稳定值文件。
10. 拔出滤光片，通道回到空气基底附近并稳定 10 帧后，允许再次插入并覆盖该通道稳定值。

稳定条件：

```text
最近 10 帧极差 < 0.005
```

30% 滤光片默认判定范围：

```text
当前值 / 空气基底 = 30% ± 5 个百分点
```

即：

```text
25% ~ 35%
```

回到空气默认判定范围：

```text
当前值 / 空气基底 = 100% ± 5 个百分点
```

即：

```text
95% ~ 105%
```

## 注意事项

- `filter_ratio` 必须在 `config.ini` 中设置为 `0/10/20/30`。
- 如果只接了一个 12 通道模块，要把 `channel_group` 设置为 `1` 或 `2`，否则空气基底阶段会等待未连接通道。
- 滤光片模式下，空气基底阶段不会写入原始数据文件；正式测试开始后才写入。
- 如果 Excel 文件被打开占用，脚本会提示先关闭文件。
- 当前数据是未经过软件校准的原始强度类读数，因此滤光片比例按 `当前值 / 空气基底` 判断。
- 如果使用 30% 滤光片，理论上滤光片后的读数接近空气基底的 30%。
